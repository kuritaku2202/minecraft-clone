**Sprint 0〜13 完了。** Sprint 13 詳細（Playwright検証済み）:
- **アイテム/インベントリ基盤**: ブロック以外のアイテム（棒・道具・防具・素材）を表す
  `items.ts`、36スロット＋防具4スロットの `Inventory.ts`、アイテムアイコン
  （ブロックは立方体、道具/防具/素材は手続き2Dスプライト）。
- **採取**: ブロック破壊でアイテムをインベントリに自動収集。石/鉱石はツルハシ必須
  （無いとドロップ無し）。モブ撃破で各種ドロップ（羊→羊毛、牛→革 等）。
- **クラフト**: 形あり/なしレシピのマッチング（`recipes.ts`）。インベントリの2×2と
  作業台の3×3。クリックでアイテム移動（右クリックで1個ずつ配置）＋リザルト回収。
  右側にレシピ一覧（クリックで一発クラフト、材料不足はグレーアウト）。
  レシピ: 板材・棒・作業台・かまど・チェスト・ベッド、木/石の剣・ツルハシ・斧・
  シャベル、革の兜/チェスト/レギンス/ブーツ。
- **装備/武器**: 防具4スロット装備で被ダメージ軽減（装甲1点≒4%、フル革で装甲7＝28%減）。
  HUDに装甲バー。剣で対モブ攻撃力アップ（素手1／木剣4／石剣5）。
- **新ブロック**: 羊毛・作業台・かまど・チェスト・ベッド（設置可・テクスチャ付き）。
  作業台を右クリックで3×3クラフトUIが開く。

Sprint 12 詳細（Playwright検証済み）:
- モブを10種に拡張（データ駆動の `MobType.ts`）。動物（豚・牛・羊・鶏＝受動・昼に草地で
  スポーン）とモンスター（ゾンビ・スケルトン・クリーパー・スパイダー・スライム・
  エンダーマン＝敵対・夜に任意の地表でスポーン）。各種に専用のブロックモデル
  （脚アニメ／鶏の羽ばたき／スライムのスカッシュ／クリーパーの膨張点滅）。
- 敵対モブは検知範囲(16)でプレイヤーを追跡し接触ダメージ。クリーパーは導火線→爆発で
  地形を球状に破壊＋プレイヤーにダメージ。ゾンビ/スケルトンは日中に日光で焼ける。
- プレイヤー体力20（ハートUI10個）、ダメージ赤フラッシュ、死亡→1.5秒で初期地点に
  リスポーン。左クリックでレイ判定によりモブを攻撃（ノックバック付き）。

Sprint 11 詳細（Playwright検証済み）:
- 徘徊するパッシブMob（豚）を実装。プレイヤーのAABB衝突ソルバ（`moveAndCollide`）を
  再利用して重力・地形衝突を処理。その上に小さな状態機械（待機/ランダム方向へ歩く）。
- 経路の基礎: 1ブロックの段差は跳んで越える、2ブロック以上の崖は手前で回避（`dropAhead`）、
  壁に詰まったら方向を再抽選。
- スポーン: 草/土/砂/雪の地表にプレイヤー周囲14〜30ブロックでスポーン（最大8体）、
  52ブロック超で自動デスポーン（チャンク描画距離の内側なので足元のチャンクは消えない）。
- 描画: ブロック状の豚モデル（胴・頭・鼻・4本脚、対角歩容の脚アニメ）。`MeshLambertMaterial`
  ＋環境光/平行光で陰影、光量は昼夜サイクルに連動（チャンクシェーダには影響しない）。

Sprint 10 詳細（Playwright検証済み）:
- ブロック種を6→15に拡張: 丸石・オーク原木（上面=年輪/側面=樹皮）・オーク板材・
  オークの葉・岩盤・砂利・石炭鉱石・鉄鉱石・雪。各タイルは手続き生成（鉱石は石
  ベースに斑点、原木は年輪/縦縞、雪は白）。タイル順は `BlockRegistry.Tile` を
  唯一の真実源とし `textures.ts` がそれを参照（16レイヤの DataArrayTexture）。
- 地形統合: y=0〜2 に岩盤の床、深部の石に鉱石/砂利の鉱脈（feature 3Dノイズ。
  既存フィールドの後に追加するため seed あたりの地形形状は不変）、標高84超の
  山頂に雪、草地に小さなオークの木（幹4〜6＋葉カノピー。チャンク内部のみに配置
  しチャンク跨ぎ書き込みを回避）。
- インベントリは全14設置可能ブロックを一覧（アイコンはタイル色から生成、各固有）。

Sprint 9 詳細（Playwright検証済み）:
- 9スロットのホットバー（下部中央）。各スロットに擬似アイソメトリックの立方体
  アイコン（上面明・側面暗のワールド同等シェーディング）を表示。選択スロットを
  白枠でハイライト。`[1]〜[9]` キー / マウスホイールで選択（巡回）。切替時に
  ブロック名がフェード表示。
- クリエイティブ風インベントリ（`[E]` でトグル、`Esc`/背景クリックで閉じる）。
  全ブロックを立方体アイコンで一覧表示。クリックで選択中スロットへ割り当て。
- ホットバーの選択スロットが「手持ちブロック」の唯一の真実源。設置はその
  ブロックを使用。空（Air）スロットでは設置不可。インベントリ開放中は
  ブロック設置/破壊を停止し、ポインターロックを解放。

Sprint 8 詳細（Playwright検証済み・PR #1 マージ済み）:
- 頂点を単一 uint32 にパック（pos5/9/5 + faceId3 + ao2 + tile8）。UV/シェードは
  シェーダ側で導出。32B/頂点 → 4B/頂点（8倍削減）。チャンクはローカル座標で
  メッシュ化し、`mesh.position` で平行移動。
- メッシュ化を WebWorker プール（最大4）にオフロード。`ChunkManager` の予算は
  「投入数」を制限。スポーン preload のみメインスレッドで同期メッシュ化。
- 水を `transparent: true` 化し、不透明パスとは別の半透明パスで水面を描画。
  水中の地形が透けて見える。

## 起動方法
```bash
npm install   # 初回のみ
npm run dev   # http://127.0.0.1:5173
```

## 検証方法
各スプリント完了時に Playwright MCP でブラウザを開き、
計画ファイルのチェックリストを確認してからコミットする。
